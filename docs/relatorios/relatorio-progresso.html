Epic 1 — Cadastro e Acesso (Fundação)

8 funcionalidades · LGPD compliant · MVP completo

✓ Concluído ▾

O que foi construído

Toda a base de cadastro e acesso do app. Antes não tinha login funcional; agora a pessoa consegue:

  1. Criar uma conta com nome, e-mail e senha — aceitando os termos da LGPD
  2. Fazer login com e-mail e senha
  3. Recuperar a senha por e-mail se esquecer
  4. Editar o perfil (nome, qual concurso quer fazer, meta de estudo, data da prova)
  5. Baixar todos os seus dados em um arquivo (direito da LGPD)
  6. Excluir a própria conta definitivamente (também LGPD)

Por trás dos panos: configuramos toda a infraestrutura — banco de dados, servidor, esteira de deploy automático. Qualquer mudança no código agora é testada e publicada sozinha.

Problemas encontrados e corrigidos

A revisão de qualidade encontrou 3 problemas de risco médio, todos resolvidos:

Brecha na exclusão de conta
Risco: alguém com acesso roubado podia tentar várias senhas para excluir conta.
Correção: bloqueio após 3 tentativas erradas em 1 hora.
Brecha de "encheção" no login
Risco: ataques de inundação travariam login da vítima.
Correção: limite de 10 tentativas por minuto por origem.
Mensagem de erro instável
Risco: atualização futura do banco podia quebrar mensagens.
Correção: detecção de erro robusta, à prova de mudanças.

Coisas aceitas como "ok por enquanto"

  • Aviso de e-mail já cadastrado: tecnicamente revela quem é cliente, mas o usuário precisa saber. Trade-off aceito.
  • Configuração manual no painel da Supabase: 1 minuto de trabalho manual.

Epic 2 — Catálogo e Seleção de Editais

4 funcionalidades · Painel admin completo · 1 deferida

✓ Concluído ▾

O que foi construído

O catálogo de editais e a forma de o usuário escolher um para estudar.

  1. Catálogo público de editais com filtros (banca, órgão, ano) e busca por nome
  2. Botão "Estudar para este edital" em cada card, que pede as datas de inscrição e prova
  3. Aviso inteligente: se já estuda outro edital, sistema avisa antes de trocar
  4. Painel admin para o Founder em /admin/editais: criar, editar, publicar, arquivar — tudo via interface
  5. Controle fino de visibilidade: público, beta, ou allowlist específica — útil para liberar gradualmente

Problemas encontrados e corrigidos

Catálogo aberto demais
Risco: API direta vazava editais sem login.
Correção: catálogo agora exige login. Reversível se decidirmos abrir para SEO.
Funções do banco com ponto cego
Risco: vetor avançado de injeção via "search path".
Correção: caminho fixado explicitamente (boa prática Supabase).
Tipagem fraca no controle de admin
Risco: mudança futura na Supabase poderia quebrar bloqueio admin sem aviso.
Correção: tipo dedicado e helper. TypeScript pega qualquer mudança no boot.

Coisas adiadas conscientemente

Story 2.5 — Múltiplos editais simultâneos: recurso premium dependente do sistema de pagamentos (Epic 7). Plano de execução documentado.

Outros adiamentos pragmáticos: publicação automática agendada, preview de markdown, tabela com filtros avançados — todos "nice to have".

Epic 3 — Banco de Questões e Fluxo de Resposta

6 funcionalidades · 1.120 questões reais · O coração do app

✓ Concluído ▾

O que foi construído

O coração do app: o usuário agora consegue de fato estudar respondendo questões com feedback inteligente.

  1. Banco de questões com 1.120 questões reais do TJSC 2026 (Técnico + Analista, banca FGV) — extraídas dos PDFs originais via parser automatizado
  2. Painel admin do curador/founder para criar, editar, publicar, arquivar questões — com histórico de revisões automático
  3. Tela "Praticar": usuário escolhe edital → recebe questão aleatória → marca alternativa → vê acerto/erro → recebe explicação
  4. Reportar problemas em questões diretamente da tela de feedback (motivo + comentário) — limite de 5 reports/dia por usuário
  5. Vínculo questão↔edital: cada questão é mostrada apenas para quem está estudando aquele edital

Problemas encontrados e corrigidos

A revisão detectou 1 problema GRAVE + 2 médios, todos resolvidos:

🚨 Brecha de trapaça no resultado das questões (GRAVE)
Risco: usuário com noção técnica podia abrir DevTools e marcar resposta errada como certa.
Correção: verificação no servidor. O banco recalcula a correção comparando com o gabarito real. Trapaça impossível.
Gabarito vazando antes da resposta
Risco: usuário podia ver alternativa correta no DevTools antes de marcar.
Correção: tela recebe questão sem gabarito. Gabarito + comentário só são baixados depois da resposta.
Bug ao trocar de edital
Risco: ao trocar de concurso, sistema continuava puxando questões do antigo.
Correção: ajuste de 1 linha na regra de acesso do banco.

Coisas aceitas como "ok por enquanto"

  • Tags livres em questões: matéria/banca/dificuldade já dão flexibilidade suficiente.
  • Curador com acesso a rascunhos: liberado só para founder. Curador entra junto com pipeline de IA (Epic 8).
  • Multi-edital simultâneo: deferido junto com o sistema de pagamentos premium.

Como foi a abordagem

Cada Epic passou pelo mesmo ciclo:

  1. Stories detalhadas com critérios de aceite claros antes de qualquer linha de código
  2. Implementação iterativa com commits granulares e testes automatizados a cada passo
  3. Revisão de qualidade independente que audita código, segurança e regras de negócio
  4. Correção dos achados antes de fechar o Epic — nada vai pra produção com pendências de risco médio ou alto
  5. Atualização do board (Monday) com status, sprint e resumo executivo em linguagem simples

Tudo está versionado em Git, com deploy automático a cada merge na branch principal.
